Mobile Project Navigation — 3 Approaches

All use swipe with thumbnail+title peek. The difference is how the nav is surfaced and what else is available.

A

Swipe-Only (Purist)

No visible nav buttons at all. Swipe left/right on the project page to peek the next/prev project card sliding in from the edge. Swipe far enough to commit, or release to snap back.

Pros

  • Maximum clean — nothing competes with content
  • Feels native, like iOS Photos
  • No bottom bar, no browser UI conflict

Cons

  • Zero discoverability — new visitors won't know they can swipe
  • No "back to grid" shortcut
  • Accessibility concerns (no tap target)
B

Swipe + Subtle Hint Bar

Same swipe-to-peek gesture, but a minimal bar at the bottom shows prev/next project titles (truncated). Tapping either side also navigates. A small grid icon in the center returns to /work/. The bar is always visible — no hide-on-scroll.

Pros

  • Discoverable — titles hint that more projects exist
  • Tap fallback for users who don't swipe
  • Grid shortcut solves the "back to portfolio" problem
  • Always visible = no disorientation

Cons

  • Still a bottom bar (though thinner and more informative)
  • Slightly more visual weight than pure swipe
C

Swipe + Edge Peek Indicators

No bottom bar. Instead, thin accent-colored edges on left/right hint at swipeable content. On first visit, a subtle one-time animation peeks the next project card 40px in from the right, then snaps back — teaching the gesture. A floating grid button (top-right, near nav) provides the back-to-grid shortcut.

Pros

  • No bottom bar at all — zero browser UI conflict
  • Onboarding animation teaches the gesture naturally
  • Edge indicators are minimal but discoverable
  • Grid button integrated with existing nav area

Cons

  • Edge indicators alone may not be enough for all users
  • One-time animation only helps first-time visitors
  • Slightly more complex to implement

My recommendation is B — it balances the premium swipe feel with real-world usability. The always-visible titles give context ("where am I in the portfolio"), the grid icon solves the back-navigation problem, and tap still works for people who don't swipe.
